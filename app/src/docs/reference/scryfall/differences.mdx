# Scryfall Differences

Frantic Search extends [Scryfall's query syntax](https://scryfall.com/docs/syntax) with additional features and some behavioral differences. This page summarizes the main divergences. Per-field articles include Scryfall notes where relevant.

## Behavioral divergences

| Behavior | Scryfall | Frantic Search |
|----------|----------|----------------|
| Field clause whitespace | Whitespace between the field name and `:` / `=` (or between the operator and the value) breaks the field clause so a new term starts | **Same clause boundaries** (Spec 002 § Whitespace and field clauses; GitHub #240). Example: `kw: otag:ramp` is two terms; the second is `otag:ramp`, not `kw` with value `otag`. |
| `field:` with no adjacent value (trailing operator) | Not documented. In practice, `kw:` without a glued value tends to fall through to **bare** behavior: the query behaves as if the text were a bare name fragment, so normalized names containing those letters can match (e.g. Beanstalk Wurm for `kw:`). | Parsed as a **`FIELD` with an empty string value**; evaluators treat that as a **neutral filter** (matches all) unless a field-specific spec says otherwise. Scryfall does not document this edge case; we chose an interpretation we consider higher-fidelity for **malformed** syntax. |
| [Bare regex](?doc=reference/special/bare-regex) | Not supported | `/pattern` searches name, oracle text, and type line |
| Bare words | Searches name (fuzzy matching) | Searches name (substring, no fuzzy). See [name](?doc=reference/fields/face/name). |
| Percentile filters | Not supported | [usd](?doc=reference/fields/printing/usd), [date](?doc=reference/fields/printing/date), [name](?doc=reference/fields/face/name), [edhrec](?doc=reference/fields/card/edhrec), [salt](?doc=reference/fields/card/salt): `usd>90%`, `date<10%`, `name>50%`, etc. |
| =null | Not supported | Nullable numeric fields support `=null` to match faces and printings with no data |
| $ alias | Uses usd | [usd](?doc=reference/fields/printing/usd): `$` is alias for usd |
| [produces](?doc=reference/fields/card/produces):t | Returns "Unknown color t" | Matches Sole Performer (T supported) |
| Query speed | Server round-trip | Instant (client-side, every keystroke) |
| [`is:alchemy`](?doc=reference/fields/face/is) | Default search omits a few MB2 playtest printings that still have the alchemy promo tag in bulk | We honor bulk `promo_types` and `set_type: alchemy`. Example: [is:alchemy set:mb2](?q=is%3Aalchemy%20set%3Amb2) |
| [`set:`](?doc=reference/fields/printing/set) | Exact set code (unknown code → 0 cards) | **Prefix** on normalized set code for discovery; empty `set:` matches printings with a set code; no matching prefix → 0 results, no leaf error |
| [`otag:`](?doc=reference/fields/card/otag) / [`atag:`](?doc=reference/fields/printing/atag) | Effectively exact tag labels | **Prefix** on normalized tag keys with **union** over all matching keys; empty `otag:` / `atag:` matches faces/printings that have any tag; no matching prefix → 0 results, no leaf error |
| [`kw:`](?doc=reference/fields/face/kw) / `keyword:` | Effectively exact keyword names | **Prefix** on normalized keyword index keys with **union** over all matching keys; empty `kw:` matches all faces; no matching prefix → **`unknown keyword`** (passthrough), unlike `set:` / `otag:` / `atag:` |
| [Power / toughness / loyalty / defense](?doc=reference/fields/face/power) | Mostly plain-numeric stat filters on Scryfall | Frantic also supports substring and exact match on oracle stat text, quoted values, and equatable-null prefixes; ranges require plain numbers. **`pow=x`** matches literal oracle **`X`**; use **`pow=0`** for numeric zero. See [power](?doc=reference/fields/face/power), [toughness](?doc=reference/fields/face/toughness), [loyalty](?doc=reference/fields/face/loyalty), [defense](?doc=reference/fields/face/defense). |

## Known missing

Some Scryfall features are not yet implemented. See [Known Gaps](?doc=reference/scryfall/gaps) for details. Examples: `new:rarity` (first printed at new rarity); partial `is:` keyword support. See [is](?doc=reference/fields/face/is).
