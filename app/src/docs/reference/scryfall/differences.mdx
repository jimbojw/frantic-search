# Scryfall Differences

Frantic Search extends [Scryfall's query syntax](https://scryfall.com/docs/syntax) with additional features and some behavioral differences.

This page has two parts:

1. **Cross-cutting divergences** — behaviors that span many fields or the whole query language. This document records them.
2. **Field and feature index** — short pointers to articles where a specific field or modifier is documented, including how it differs from Scryfall.

For features we have not implemented, see [Known Gaps](?doc=reference/scryfall/gaps).

## Cross-cutting divergences

### Field clause whitespace

Whitespace between the field name and `:` or `=` (or between the operator and the value) **breaks** the field clause so a **new term** starts. Frantic matches Scryfall on these clause boundaries.

Example: `kw: otag:ramp` is **two** terms. The second is `otag:ramp`, not `kw` with value `otag`.

If you **did** mean a single field clause (e.g. `ci: blue` intending `ci:blue`) but typed a space after the operator, a zero-result search may show a **Try a query refinement?** chip that removes the gap when that rewrite returns cards.

### Trailing operator (no adjacent value)

When `field:` has **no value token glued** to the operator (whitespace or end of input), Scryfall does **not** document the behavior. In practice, something like `kw:` often **falls through** to **bare** behavior: the query behaves like a bare name fragment, so normalized names containing those characters can match (e.g. Beanstalk Wurm for a lone `kw:`).

Frantic parses that shape as a **`FIELD` with an empty string value**. Evaluators treat that as a **neutral filter** (matches all) unless a field-specific spec says otherwise. We treat this as a clearer reading of **malformed** syntax where Scryfall leaves behavior implicit.


### Bare regex

Scryfall does not support a bare `/pattern/` term. Frantic searches **name**, **oracle text**, and **type line** with one regex. See [Bare regex](?doc=reference/special/bare-regex).

### Percentile filters

Scryfall does not support percentile syntax. Frantic supports it on several fields, for example `usd>90%`, `date<10%`, `name>50%`. Each field’s article describes operators and behavior:

[usd](?doc=reference/fields/printing/usd), [date](?doc=reference/fields/printing/date), [name](?doc=reference/fields/face/name), [edhrec](?doc=reference/fields/card/edhrec), [salt](?doc=reference/fields/card/salt).

### `=null` on nullable fields

Scryfall does not document `=null` for query values. Frantic supports `=null` (and related null-literal forms where specified) on nullable numeric and other fields so you can match faces or printings with no data. See individual field docs (e.g. mana, stats, printing price fields).

### `$` as `usd`

Scryfall uses `$` as an alias for USD price. Frantic does the same. See [usd](?doc=reference/fields/printing/usd).

### Query speed and architecture

Scryfall evaluates queries on the server. Frantic runs the full pipeline **in the browser** (Web Worker + card index) so search updates on every keystroke without a round-trip.

## Field and feature index

Use these links for **Scryfall vs Frantic** detail on a single field or keyword.

| Topic | Documentation |
|-------|----------------|
| [`produces:`](?doc=reference/fields/card/produces) | Mana-production filter; Frantic supports **T** where Scryfall may report an unknown color. |
| [`is:alchemy`](?doc=reference/fields/face/is) and alchemy tagging | Bulk `promo_types` / set type behavior vs Scryfall’s default search omissions; example query [is:alchemy set:mb2](?q=is%3Aalchemy%20set%3Amb2). |
| [`set:`](?doc=reference/fields/printing/set) | Scryfall targets an **exact** set code; Frantic **`:`** uses **prefix** on the normalized code for discovery and **`=`** uses **normalized exact** match (escape hatch). Empty **`=`** is neutral; empty **`:`** still requires a non-empty stored code; **unknown non-empty** value under the active operator → **`unknown set`** (passthrough). |
| [`frame:`](?doc=reference/fields/printing/frame) | Scryfall treats **`:`** / **`=`** as exact-oriented; Frantic **`:`** uses **normalized prefix** over the closed vocabulary (ORs eras for discovery) and **`=`** uses **normalized exact** match. **`frame!=`** is a **Frantic extension** (negates **`frame=`** only, like **`in!=`**); Scryfall does not document it. Empty **`frame:`** / **`frame=`** / **`frame!=`** are neutral (all printings). |
| [`set_type:`](?doc=reference/fields/printing/set_type) / [`st:`](?doc=reference/fields/printing/set_type) | Scryfall targets an **exact** set type token; Frantic **`:`** uses **prefix** on the normalized type and **`=`** uses **exact** (same split as `set:` / `kw:`). **Unknown non-empty** value → **`unknown set_type`** (passthrough). |
| [`otag:`](?doc=reference/fields/card/otag) / [`atag:`](?doc=reference/fields/printing/atag) | Effectively exact tag labels on Scryfall vs **prefix** + union over keys in Frantic; unknown non-empty prefix → **zero hits, no leaf error** (unlike `set:` / `set_type:` / `kw:`). |
| [`kw:`](?doc=reference/fields/face/kw) / `keyword:` | Effectively exact keyword names on Scryfall vs Frantic **`:`** = **normalized prefix** + union over index keys, **`=`** = **normalized exact** match (escape hatch). Empty value matches all faces; **unknown non-empty** value under that operator → **`unknown keyword`** passthrough—aligned with `set:` / `set_type:`, not with tag fields. |
| [`is:`](?doc=reference/fields/face/is) / [`not:`](?doc=reference/fields/face/is) | Scryfall treats `is:` values as effectively **exact** tokens; Frantic matches a **normalized prefix** against the closed keyword vocabulary and **unions** all matches (e.g. a short prefix can match both a face-level and a printing-level keyword). A non-empty prefix that matches **no** keyword yields **`unknown keyword`** with query **passthrough** (like `kw:`), not a silent empty result. |
| [power](?doc=reference/fields/face/power), [toughness](?doc=reference/fields/face/toughness), [loyalty](?doc=reference/fields/face/loyalty), [defense](?doc=reference/fields/face/defense) | Scryfall exhibits mostly plain-numeric stat filters; Frantic adds substring and exact match on oracle stat text, quoted values, and equatable-null forms. |
| [Sort by set](?doc=reference/sorting/sort-fields) (`sort:set`) | Frantic sorts by lexicographic set code and maps Scryfall outlinks to `order=set` / `dir=`. |

## Known missing

Some Scryfall features are not yet implemented. See [Known Gaps](?doc=reference/scryfall/gaps). Examples: `new:rarity`; partial `is:` keyword coverage — see [is](?doc=reference/fields/face/is).
