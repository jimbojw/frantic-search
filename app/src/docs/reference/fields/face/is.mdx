# is

Mechanics, layouts, roles, and finish (e.g. commander, dfc, foil). Keywords mix oracle/layout traits with printing traits (foil, frame, promo); see the lists below for which is which.

**Canonical:** `is`  
**Aliases:** `not`  
**Domain:** Face (printing-only keywords still use printing data in the evaluator)

## Operators

| Operator | Behavior | Example |
|----------|----------|---------|
| `:` | Has property | `q=is:commander` |
| `=` | Has property | `q=is:dfc` |

## Convenience: not:

The `not:` field is a Scryfall convenience for `-is:`. It uses the same keywords and operators.

| Query | Equivalent to |
|-------|---------------|
| `q=not:dfc` | `q=-is:dfc` |
| `q=-not:dfc` | `q=is:dfc` |

## Prefix discovery and union

Frantic resolves your value as a **prefix** over the **closed list** of supported `is:` keywords (face-level and printing-level names, including aliases such as `ub` for `universesbeyond`). **Every** keyword whose normalized form starts with your prefix contributes matches; results are **OR**’d. So one term can combine several keywords at once—printing-only keywords (e.g. foil, atypical) still run in the **printing** domain when printings are loaded, and face-only keywords run on oracle faces; mixed prefixes combine after promotion.

- **Empty** `is:` / `not:` (nothing after the operator, after trim) matches **all** rows in that leaf’s domain—same idea as empty [`kw:`](?doc=reference/fields/face/kw).
- A **non-empty** prefix that matches **no** keyword produces an **`unknown keyword`** leaf error; other query terms can still apply (**passthrough**).

On Scryfall, `is:` values behave like **exact** (or near-exact) tokens. Prefix union here is intentional for discovery; see [Scryfall Differences](?doc=reference/scryfall/differences).

## Allowed values

Keywords for mechanics, layouts, roles, and finish. Examples:

| Category | Examples |
|----------|----------|
| Mechanics | `q=is:commander`, `q=is:legendary`, `q=is:vanilla` |
| Layouts | `q=is:dfc`, `q=is:meld`, `q=is:split` |
| Roles | `q=is:booster`, `q=is:promo` |
| Finish | `q=is:foil`, `q=is:etched`, `q=is:nonfoil` |
| Frame | `q=is:old`, `q=is:new` (era); `q=is:default`, `q=is:atypical` (treatment) |
| Land cycles | `q=is:fetchland`, `q=is:shockland`, `q=is:pathway` |
| Promo / printing (when printings load) | `q=is:alchemy`, `q=is:rebalanced`, `q=is:rainbowfoil`, … | See below |

Matching is case-insensitive. See the [Syntax cheat sheet](?doc=reference/syntax) for the full list.

## `is:alchemy` (printing-level)

`is:alchemy` matches printings where Scryfall bulk data marks the card as Alchemy: either **`promo_types` includes `alchemy`** or the printing’s **`set_type` is `alchemy`** (e.g. Alchemy: Outlaws of Thunder Junction). Frantic Search follows that data rather than hiding printings that scryfall.com search omits.

For a small edge case, **seven** [Mystery Booster 2](?q=set%3Amb2+unique%3Aprints+include%3Aextras) playtest printings carry the alchemy promo tag in bulk but do not appear in Scryfall’s default search for `is:alchemy`. Frantic includes them. To see exactly those printings, run **[is:alchemy set:mb2](?q=is%3Aalchemy%20set%3Amb2)**.

## Notes

- `q=is:commander` matches cards that can be your commander (legendary creatures, planeswalkers with "can be your commander", etc.).
- `q=is:old` and `q=is:new` match printings by **frame era** (classic 1993/1997 vs post-classic 2003/2015/future), using the same `frame` data as `frame:`. They are not the same as **`is:default`** / **`is:atypical`**, which classify atypical frame *treatments* (borderless, showcase, etc.).
- `q=is:default` matches printings with the default Magic frame; `q=is:atypical` matches printings with atypical frame treatments (borderless, extended art, showcase, etc.).
- Land cycle keywords (e.g. `q=is:fetchland`) match curated card-name sets.

## Scryfall

Frantic Search supports a subset of Scryfall's `q=is:` keywords. Some values (e.g. `q=new:rarity` for "first printed at new rarity") are not yet implemented. Prefix + union behavior (above) also differs from Scryfall’s effectively exact `is:` tokens—see [Scryfall Differences](?doc=reference/scryfall/differences).
