# otag

Community-curated tags describing card function (e.g. ramp, removal).

**Canonical:** `otag`  
**Aliases:** `function`, `oracletag`  
**Domain:** Card

## Operators

| Operator | Behavior | Example |
|----------|----------|---------|
| `:` | Boundary-aligned prefix match on tag-normalized keys (union) | `q=otag:ramp` |
| `=` | Exact match on the full tag-normalized key | `q=otag=ramp` |
| `!=` | Negates exact `=` only (Frantic extension) | `q=otag!=removal` |

## Matching

Tag values use **tag normalization** (accents folded, lowercase, **hyphens preserved**; other punctuation stripped)—not the same alphanumeric-only normalization as fields like `set:` or `kw:`. Hyphens mark **word boundaries** in hyphenated slugs.

- **Prefix (`:`):** A tag **matches** when the normalized query is a **prefix** of the normalized tag starting at the **beginning of the slug** or **immediately after a hyphen** (boundary-aligned prefix). Results are the **union** (OR) of all matching tags. Examples: `otag:mana-r` matches `mana-ramp`; `otag:trigger` matches `death-trigger`; `otag:ana` does not match `mana`; `otag:on-` does not spuriously match unrelated tags that only shared a stripped `on` prefix.
- **Exact (`=`):** Only keys whose **full** tag-normalized form **equals** the query match (e.g. `otag=ramp` does not match a sole `ramp-artifact` key).
- **`!=`:** Matches faces that do **not** carry the exact tag from `=` (use `-otag:…` / `NOT` to exclude a `:` union).
- **Empty value:** `otag:` or `otag=` with nothing after the operator matches every **face** that has **at least one** oracle tag in the loaded index (narrower than empty `kw:`; see [Scryfall differences](?doc=reference/scryfall/differences)).
- **Empty `!=`:** Inverts that “any tag” set—faces with **no** oracle tags in the loaded index.
- **No match:** Non-empty value with no key matching under the active operator → **`unknown oracle tag "…"`** with query passthrough.

**Canonical Scryfall links** may still expand a uniquely resolvable short value via auto-resolution; in-app search uses the operator rules above. See [set](?doc=reference/fields/printing/set) for the same `:` / `=` split shape on another field (with different normalization).

## Notes

Oracle tags come from the tag data model; see [tag-filter](?doc=reference/special/tag-filter) for `#tag` filtering.
