# set

Set code (e.g. mh2, dmu).

**Canonical:** `set`  
**Aliases:** `s`, `e`, `edition`  
**Domain:** Printing

## Operators

| Operator | Behavior | Example |
|----------|----------|---------|
| `:` | Prefix match on normalized set code (discovery) | `q=set:mh2`, `q=set:u` |
| `=` | Exact match on normalized set code | `q=set=mh2` |

Comparison operators (`>`, `<`, `!=`, …) are not supported on `set:`.

## Matching

Values are normalized the same way as other categorical fields (lowercase, alphanumeric only): `set:9ed` and set codes that normalize the same way prefix-match together.

- **Prefix (`:`):** Each printing matches if its normalized set code **starts with** the normalized query (e.g. `set:u` matches `ugin`, `usd`, …).
- **Exact (`=`):** Each printing matches only if its normalized set code **equals** the normalized query (e.g. `set=sl` does **not** match `sld` / `slb`).
- **Empty value:** `set:` with nothing after the operator matches every printing whose normalized set code is **non-empty**. **`set=`** with an empty value is **neutral** (does not narrow results).
- **No match:** If no printing matches under **`:`** (prefix) or **`=`** (exact), the leaf returns **`unknown set "…"`** (trimmed value). Scryfall may show 0 cards for an impossible token instead of a named error.

**`:`** is tuned for **discovery**; **`=`** matches Scryfall-style exact codes on the normalized form. See [Scryfall differences](?doc=reference/scryfall/differences).

## Notes

- Set codes follow Scryfall's [set list](https://scryfall.com/sets).
- For "cards printed in set X", combine with face filters (e.g. type line): `q=t:creature set:mh2`.
- Canonical Scryfall links may still expand a uniquely resolvable short `set` value to the full code; in-app search uses prefix matching on printings. See [Scryfall differences](?doc=reference/scryfall/differences).
