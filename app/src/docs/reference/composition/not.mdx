# NOT

Prefix a term with `q=-` to exclude cards that match it.

## Syntax

| Query | Meaning |
|-------|---------|
| `q=-c:black` | Cards that are not black |
| `q=-t:creature` | Cards with no creature face |
| `q=-usd>100` | Printings with price ≤ $100 (or no price data) |

## Leaf-aware negation

For comparison operators, NOT uses **operator inversion** instead of simple exclusion. This ensures null/missing values are excluded from the negated result.

| Negated term | Equivalent to | Null handling |
|--------------|--------------|---------------|
| `q=-usd>10` | `q=usd<=10` | Both exclude printings with no price data |
| `q=-pow>=4` | `q=pow<4` | Both exclude non-creatures (no power) |
| `q=-tou>5` | `q=tou≤5` | Both exclude cards without toughness |

So `q=-usd>10` ⟺ `q=usd<=10` — the negated term excludes nulls, matching mathematical expectation.

For equality (`q==`, `q=!=`), negation uses normal buffer inversion. Example: `q=-usd=null` matches printings that have price data.

## Scryfall

Scryfall does not support negated price queries. A query like `q=-usd<2000` on Scryfall produces a text search for `q="usd<2000"` exclusion, not a price filter. Frantic Search's leaf-aware negation is a principled divergence.
