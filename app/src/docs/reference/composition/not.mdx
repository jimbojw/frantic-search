# NOT

Prefix a term with <QueryExample query="-"/> to exclude cards that match it.

## Syntax

| Query | Meaning |
|-------|---------|
| `-c:black` | Cards that are not black |
| `-t:creature` | Cards with no creature face |
| `-usd>100` | Printings with price ≤ $100 (or no price data) |

## Leaf-aware negation

For comparison operators, NOT uses **operator inversion** instead of simple exclusion. This ensures null/missing values are excluded from the negated result.

| Negated term | Equivalent to | Null handling |
|--------------|--------------|---------------|
| `-usd>10` | `usd<=10` | Both exclude printings with no price data |
| `-pow>=4` | `pow<4` | Both exclude non-creatures (no power) |
| `-tou>5` | `tou≤5` | Both exclude cards without toughness |

So <QueryExample query="-usd>10"/> ⟺ <QueryExample query="usd<=10"/> — the negated term excludes nulls, matching mathematical expectation.

For equality (<QueryExample query="="/>, <QueryExample query="!="/>), negation uses normal buffer inversion. Example: <QueryExample query="-usd=null"/> matches printings that have price data.

## Scryfall

Scryfall does not support negated price queries. A query like <QueryExample query="-usd<2000"/> on Scryfall produces a text search for <QueryExample query={'"usd<2000"'}/> exclusion, not a price filter. Frantic Search's leaf-aware negation is a principled divergence.
