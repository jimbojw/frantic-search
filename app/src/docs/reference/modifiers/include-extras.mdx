# include:extras

**Canonical:** `q=include:extras`  
**Alias:** `q=**` (Frantic Search–exclusive)  
**Domain:** Face (modifier; applies to entire query)

## Purpose

By default, Frantic Search excludes non-playable cards and non-tournament-usable printings from search results — matching Scryfall's behavior. The `q=include:extras` modifier bypasses this filter and shows all matches, including acorn-stamped cards, silver-bordered cards, playtest cards, gold-bordered printings, oversized, and 30th Anniversary Edition.

## Syntax

| Form | Example |
|------|---------|
| `q=include:extras` | `q=t:creature include:extras` |
| `q=**` | `q=t:bolt **` |

The bare word `q=**` desugars to `q=include:extras`. It behaves identically to the canonical form. Desugared nodes carry `sourceText` so the breakdown displays the original token.

## Behavior

- `q=include:extras` is a query modifier, not a filter. It follows the same pattern as `q=unique:prints` — it produces a match-all buffer and does not affect which cards match.
- When present, the evaluator sets `includeExtras: true` on the output. The worker skips the default playable filter entirely.
- When absent, the worker applies the playable filter: cards must be legal or restricted in at least one format; printings must be tournament-usable (no gold-bordered, oversized, etc.).

## Examples

- `q=t:creature **` — All creatures, including non-playable ones
- `q=f:commander include:extras` — Commander-legal cards plus extras (e.g. acorn-stamped)

## Scryfall

Scryfall uses `q=include:extras` with the same semantics. The `q=**` alias is Frantic Search–exclusive.
